笔记｜Flex用法

Sep 11 2022

查阅flex用法

属性

flex 弹性布局，是一种布局的形式，故有容器框和被包裹内容两种元素，称为 Flex容器（container） 和 Flex项目（item）。它们分别有各自属性。

容器 container 属性：

  • display: flex // 设为flex

// 决定主轴的方向

  • flex-direction： row | row-reverse | column | column-reverse

// 如果一条轴线排不下，如何换行

  • flex-wrap：nowrap ｜ wrap ｜ wrap-reverse

// flex-direction和flex-wrap 合体

  • flex-flow：<flex-direction> || <flex-wrap>

// 项目在主轴上的对齐方式

  • justify-content
    • flex-start（默认值）：左对齐
    • flex-end：右对齐
    • center： 居中
    • space-between：两端对齐顶着边儿，项目之间的间隔都相等。
    • space-around：每个项目两侧的间隔相等。所以，项目之间的间隔比项目与边框的间隔大一倍

// 项目在交叉轴上如何对齐

  • align-items
    • flex-start：交叉轴的起点对齐。
    • flex-end：交叉轴的终点对齐。
    • center：交叉轴的中点对齐。
    • baseline: 项目的第一行文字的基线对齐。
    • stretch（默认值）：如果项目未设置高度或设为auto，将占满整个容器的高度。

// 多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用。

  • align-content
    • flex-start：与交叉轴的起点对齐。
    • flex-end：与交叉轴的终点对齐。
    • center：与交叉轴的中点对齐。
    • space-between：与交叉轴两端对齐，轴线之间的间隔平均分布。
    • space-around：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍。
    • stretch（默认值）：轴线占满整个交叉轴。

项目的属性

// 定义项目的排列顺序。数值越小，排列越靠前，默认为0。

  • order

// 项目的放大比例，默认为0。
// 0表示如果存在剩余空间，也不放大。数字越大越优先放大。

  • flex-grow

// 定义了项目的缩小比例，默认为1。
// 1 表示如果空间不足，该项目将缩小。0表示不缩小。

  • flex-shrink

// 在分配多余空间之前，项目占据的主轴空间。
// 默认值为auto，即项目的本来大小。
// 可以设为跟width或height属性一样的值（like350px），则项目将占据固定空间。

  • flex-basis

//flex-grow, flex-shrink 和 flex-basis的简写。推荐使用flex缩写语法
// 默认值为0 1 auto

  • flex
    • flex:initial 即 0 1 auto。不扩展，收缩，自适应。
      适合小控件元素的分布布局，或者某一项内容动态变化的布局；
    • flex:0 即 0 1 0%。 不扩展，收缩，最小内容宽度。
      适用场景较少，适合设置在替换元素的父元素上；
    • flex:none 即 0 0 auto。不扩展，不收缩，自适应。
      适用于不换行的内容固定或者较少的小控件元素上，如按钮；
    • flex:1 即 1 1 0%。扩展，也收缩，尺寸不足时会优先最小化内容尺寸
      适合等分布局；
    • flex:auto 即 1 1 auto。扩展，也收缩，尺寸不足时会优先最大化内容尺寸
      适合基于内容动态适配的布局；

// 允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性。默认值为auto，表示继承父元素的align-items属性，如果没有父元素，则等同于stretch。

  • align-self ： auto | flex-start | flex-end | center | baseline | stretch;

其他注意点

  • position的absolute和fixed不参与 flex 布局
