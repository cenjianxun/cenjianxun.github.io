CAP定理

Mar 21 2021

Consistency，Availability，Partition tolerance 不可能同时做到。

consistency：同一时间访问所有节点的数据都是一致的
availability：（即使部分故障）系统能否总响应的了client的读写请求
partition tolerance：就是分布式的前提，一定是分区的

假设场景：
client A：→ 写：将v0变为v1 → server A：v0v1
client B：→ 读 → server A backup：v0
⇒
此时面临两个选择：
① AP without C：保证可用性，牺牲一致性。返回v0给B
② CP without A：保证一致性，牺牲可用性。先同步再返回v1给B

分布式系统设计的两大原则

  1. 通过复制提高可用性
    主动复制：用原子广播保证一致性及顺序
    缺点：大多数真实server都是非确定性
    被动复制：主从。可用于非确定性过程
    缺点：失败时，客户端响应会被延迟。

    同步复制：（原子）完全写入
    异步复制：性能高，最近数据可能丢失

    复制技术三大模型：
    ① 事务复制 ② paxos复制 ③ 虚拟同步
    ① 主从复制 ② 环形复制 ③ 链式复制

  2. 使用CAP理论知道分布式系统设计
    P是自然事实，CA是强需求，三者不对等

    Serializability ⇤ C (ACID) ← CAP → A (base) ⇥ Eventual Consistency

    具体场景：
    【多】AP：不涉及钱 松散的。抢票、微博。couchDB cassandra dynamo
    【少】CP：涉及钱的 精确的。银行、订票。mongoDB HBase Zookeeper
